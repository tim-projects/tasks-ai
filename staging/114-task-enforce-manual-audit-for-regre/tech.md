The 'Rc' (RegressionCheck) flag in the task metadata acts as the single source of truth for the gate. Currently, 'tasks modify --regression-check' simply toggles this flag. We need to introduce an 'audit' metadata record, possibly cryptographically signed or at least verified via a unique task-specific token, generated only after a human user has interacted with the 'repo audit' command. The 'tasks' state machine must be updated to require this audit record before allowing transition to STAGING.
