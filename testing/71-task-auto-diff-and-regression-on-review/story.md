As a developer, I want that when a task is moved to REVIEW, the system automatically creates a diff file (against main branch, including unstaged changes) in the .tasks/worktree. The agent should be instructed to check this diff for code regressions. If regressions are found, the task must be moved back to an earlier state (PROGRESSING or TESTING) for fixes. An additional checkbox (Rc - regression check passed) must be confirmed before allowing REVIEW → STAGING transitions. Code with regressions should not remain in REVIEW state.
